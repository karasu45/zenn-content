## Nerve とは

**Nerve** は、個人開発者のための統合モニタリングプラットフォームです。

死活監視、インシデント管理、セキュリティスキャン、公開ステータスページを**1つのサービスで**提供します。Free tierがあるので、まずは無料で試せます。

[NerveのURL]

## なぜ作ったか

個人開発で複数のWebサービスを運用する中で、監視周りのツールが散らばっている問題にずっと悩んでいた。

- 死活監視: UptimeRobot
- インシデント管理: スプレッドシート（手動）
- ステータスページ: 別サービス
- セキュリティチェック: 手動でたまに実行

ダッシュボードは4つ、通知はバラバラ、設定は二重管理。月額も合計すると結構かかる。

「全部入りで安くて、個人開発者に必要十分なツール」が欲しかった。探してみたけど見つからなかったので、自分で作った。

## 機能紹介

### HTTP死活監視

URLを登録するだけで監視が始まる。

- チェック間隔: 30秒〜10分（プランによる）
- ステータスコード、レスポンスタイム、SSL証明書をチェック
- レスポンスタイムのグラフ表示

[スクリーンショット: モニター一覧画面]

### 自動インシデント管理

3回連続でチェックが失敗すると、自動的にインシデントを生成。回復したら自動でクローズ。

- タイムラインで経緯を追える
- 手動でのインシデント作成も可能
- 「いつ落ちて、いつ復旧して、何分間ダウンしていたか」が自動記録

[スクリーンショット: インシデントタイムライン]

### 通知

Email、Slack、Discordに対応。モニターごとに通知先を設定できる。

- インシデント発生時に即座に通知
- 回復時にも通知
- Slack/DiscordはWebhook URLを貼るだけ

[スクリーンショット: 通知チャンネル設定]

### セキュリティスキャン（ASM）

SSL証明書、セキュリティヘッダー（6項目）、DNS設定（SPF/DMARC/CAA）を自動チェック。

- A〜Fのグレードで評価
- カテゴリ別のブレイクダウン表示
- 改善ポイントの提示

[スクリーンショット: セキュリティグレード]

### 公開ステータスページ

プロジェクトごとに公開ステータスページを自動生成。

- 各モニターの現在のステータス
- 稼働率の表示
- ユーザーに共有可能なURL

[スクリーンショット: ステータスページ]

## 技術スタック

- Next.js 16 + TypeScript + Tailwind CSS 4
- Supabase (PostgreSQL + Auth + RLS)
- QStash (監視エンジンのFan-out)
- Resend (メール通知)
- Stripe (課金)
- Vercel (デプロイ)

技術的な詳細は、別記事「個人開発者が月$9で使える統合モニタリングの作り方」で公開しています。
