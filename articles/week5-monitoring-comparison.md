## はじめに

個人開発でWebサービスを公開したとき、「ちゃんと動いてるかな」と気になったことはありませんか？

自分は以前、公開していたサービスが3日間ダウンしているのに気づかなかったことがある。ユーザーからの問い合わせで初めて知った。あれ以来、死活監視は必ず入れるようにしている。

でも「監視ツール」で検索すると、Datadog、New Relic、PagerDuty…企業向けの高機能（高価格）なツールばかり出てくる。個人開発者が月数千円〜数万円払うのは現実的じゃない。

そこで、**無料プランで使える監視ツール5つを実際に試して比較**してみた。

## そもそも「サービス監視」って何をするの？

最低限やりたいのは**死活監視（Uptime Monitoring）**。一定間隔でURLにHTTPリクエストを送り、正常にレスポンスが返ってくるかをチェックする。

チェック項目は主に：
- ステータスコードが200か
- レスポンスタイムが閾値以内か
- レスポンスボディに特定の文字列が含まれるか（キーワードチェック）

異常を検知したら、メールやSlackで通知してくれる。これがあるだけで「落ちてるのに気づかない」問題は解消される。

## 比較した5つのツール

### 1. UptimeRobot

最も有名な無料監視ツール。

| 項目 | 無料プラン |
|------|-----------|
| モニター数 | 50 |
| チェック間隔 | 5分 |
| 通知 | メール |
| ステータスページ | あり（無料） |
| SSL監視 | あり |

**良い点**: 無料で50モニターは太っ腹。UIもシンプルで迷わない。ステータスページも無料で使える。

**気になる点**: チェック間隔が5分（以前は無料で5分だったが、最近は変動あり）。Slack通知は有料プラン。アラートのカスタマイズ性が低い。

### 2. Betterstack (Better Uptime)

モダンなUIが特徴の監視ツール。

| 項目 | 無料プラン |
|------|-----------|
| モニター数 | 5 |
| チェック間隔 | 3分 |
| 通知 | メール、Slack |
| ステータスページ | あり（無料） |
| インシデント管理 | あり（基本機能） |

**良い点**: UIが洗練されている。Slack通知が無料で使える。インシデント管理が統合されている。

**気になる点**: 無料で5モニターは少ない。チーム機能は有料。

### 3. Checkly

開発者フレンドリーな監視ツール。APIモニタリングに強い。

| 項目 | 無料プラン |
|------|-----------|
| モニター数 | 5（APIチェック） |
| チェック間隔 | 10分 |
| 通知 | メール、Slack |
| ブラウザチェック | なし（有料） |
| IaC対応 | あり |

**良い点**: Playwrightベースのブラウザチェックが可能（有料）。Terraformで設定をコード管理できる。マルチリージョンチェック。

**気になる点**: 無料の5チェックでは足りない。ブラウザチェックは有料。個人開発には少しオーバースペック。

### 4. Pingdom

老舗の監視ツール。SolarWindsが運営。

| 項目 | 無料トライアル |
|------|--------------|
| モニター数 | 制限あり |
| チェック間隔 | 1分〜 |
| 通知 | メール、SMS |
| RUM | あり |

**良い点**: 実績と信頼性。リアルユーザーモニタリング（RUM）対応。グローバルな監視ポイント。

**気になる点**: 無料プランがない（14日トライアルのみ）。最安プランでも月$15。個人開発にはコスパが悪い。

### 5. Freshping

Freshworksの無料監視ツール。

| 項目 | 無料プラン |
|------|-----------|
| モニター数 | 50 |
| チェック間隔 | 1分 |
| 通知 | メール |
| ステータスページ | あり |
| マルチロケーション | 10拠点 |

**良い点**: 無料で50モニター＆1分間隔は最強クラス。10拠点からのマルチロケーションチェック。

**気になる点**: UIが少し古い。Slack通知が無料では不可。カスタマイズ性が限定的。Freshworksアカウントが必要。

## 結局どれを使うべきか

**個人開発でまず1つ選ぶなら**: UptimeRobot か Freshping。無料で50モニター使えるのは大きい。

**UIの良さとSlack通知を重視するなら**: Betterstack。ただし5モニターの制約がある。

**正直な感想**: どのツールも「死活監視」単体なら十分使える。でも、実際に運用してみると「監視だけじゃ足りない」ことに気づく。

- 障害が起きたとき → **インシデント管理**が欲しい（いつ起きた？何が原因？いつ復旧した？）
- ユーザーに状況を伝えたい → **ステータスページ**が欲しい
- セキュリティも気になる → **SSL期限切れ監視**や**セキュリティヘッダーチェック**が欲しい
- 全部の通知を一元管理したい → 複数ツールのアラートがバラバラに来るのは辛い

結局、監視ツール＋インシデント管理＋ステータスページ＋セキュリティチェックを**別々のサービスで契約する**ことになりがち。それぞれの無料枠を組み合わせれば$0で済むかもしれないけど、管理コスト（ダッシュボードの切り替え、設定の二重管理）は増える。

## まとめ

まずは無料ツールで監視を始めるのが大事。「落ちてるのに気づかない」は個人開発の信頼を一瞬で壊す。

1. **最初の一歩**: UptimeRobotかFreshpingで主要なURLを監視
2. **通知設定**: メールだけでなく、Slackやデスクトップ通知も設定
3. **ステータスページ**: ユーザー向けの状況表示を公開
